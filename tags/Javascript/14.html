GIT 명령어

Posted 5 hours Updated 5 hours
By 베짱이가 되고싶은 개미。

설정 및 설정 💡

📝 모든 커밋에서 사용할 사용자 이름과 사용할 이메일 주소를 설정합니다.

git config --global user.name "your name"
git config --global user.email "your email address"

프로젝트 시작하기 💡

📝 새로운 로컬 저장소를 초기화합니다.

git init "project-name"

📝 기존 프로젝트와 버전 이력을 복제합니다.

git clone "git url"

변경 및 스테이징 💡

📝 작업 디렉터리에서 수정된 파일의 상태를 보여줍니다.

git status

📝 파일을 스테이징 영역에 추가합니다.

git add "file"

📝 모든 수정된 파일을 스테이징 영역에 추가합니다.

git add .

📝 설명적인 메시지와 함께 스테이징된 파일을 커밋합니다.

git commit -m"commit your message"

커밋 관리 💡

📝 마지막 커밋을 수정합니다.

git commit --amend

📝 커밋 메시지를 변경하지 않고 마지막 커밋을 수정합니다.

git commit --amend --no-edit

📝 압축된 커밋 이력을 표시합니다.

git log --oneline

📝 커밋 이력을 그래픽 형식으로 표시합니다.

git log --graph

📝 다른 기본 팁 위에 변경 사항을 다시 적용합니다.

git rebase "your branch"

📝 지정된 변경 사항을 취소하는 새로운 커밋을 만듭니다.

git revert "your commit"

브랜치 및 병합 💡

📝 저장소에 있는 모든 브랜치를 나열합니다.

git branch

📝 새로운 브랜치를 생성합니다.

git branch "branch name"

📝 지정된 브랜치로 전환합니다.

git checkout "branch name"

📝 지정된 브랜치의 이력을 현재 브랜치에 병합합니다.

git merge "branch name"

📝 지정된 브랜치를 삭제합니다.

git branch -d "branch name"

프로젝트 시작 및 업데이트 💡

📝 로컬 브랜치 커밋을 원격 저장소 브랜치에 업로드합니다.

git push "alias" "branch name"

📝 원격 브랜치 변경 사항을 현재 로컬 브랜치에 통합합니다.

git pull

검사 및 비교 💡

📝 현재 브랜치의 버전 이력을 표시합니다.

git log"

📝 특정 파일의 변경 사항을 표시합니다.

git log --follow "file name"

📝 두 브랜치 사이의 콘텐츠 차이를 표시합니다.

git diff "branchB"

특정 커밋 사용하기 💡

📝 지정된 커밋의 메타데이터와 콘텐츠 변경 사항을 표시합니다.

git show "commit"

📝 지정된 커밋에서 소개된 변경 사항을 현재 브랜치에 적용합니다.

git cherry-pick "commit"

📝 현재 브랜치 팁을 지정된 커밋 위로 이동합니다.

git reset --soft "commit"

📝 현재 브랜치 팁을 지정된 커밋으로 이동합니다. 변경 사항을 스테이징하지 않습니다.

git reset "commit"

📝 작업 디렉터리를 지정된 커밋으로 재설정하고 모든 변경 사항을 삭제합니다.

git reset --hard "commit"

변경 사항 되돌리기 💡

📝 지정된 커밋 이후의 모든 커밋을 취소하고 변경 사항을 로컬로 유지합니다.

git reset "commit"

📝 지정된 커밋 이후의 모든 변경 사항을 지웁니다.

git reset --hard "commit"
