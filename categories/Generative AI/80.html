[TECH-QA] TypeScript Record<string, number> 기본

Posted 8 days Updated 8 days
By 베짱이가 되고싶은 개미。
  • Record 유틸리티 타입은 TypeScript에서 제공하는 내장 타입으로, 특정 타입의 키와 값으로 구성된 객체를 정의할 때 사용됩니다
  • 키(key): string 타입 (예: "age", "score", "id" 등).
  • 값(value): number 타입 (예: 42, 100, 3.14 등).
  • 즉, 모든 키가 문자열이고, 모든 값이 숫자인 객체를 나타냅니다.
const scores: Record<string, number> = {
  math: 95,
  science: 88,
  history: 92
};'
위 객체는 Record 타입에 부합합니다. 키는 math, science, history (모두 string)이고, 값은 95, 88, 92 (모두 number)입니다.

특징

  • 동적 키 허용: 키가 고정되지 않고, 문자열인 어떤 키든 가질 수 있습니다.
  • 값 타입 강제: 모든 값은 number 타입이어야 하며, 다른 타입(예: string, boolean)을 넣으면 컴파일 에러가 발생합니다.
const invalid: Record<string, number> = {
  name: "John" // 에러: "John"은 number가 아님
};
  • 빈 객체 가능: Record 타입의 객체는 속성이 없어도 유효합니다.
const empty: Record<string, number> = {};

다른 타입과의 비교

  • { [key: string]: number } 와 동일: Record는 인덱스 시그니처 { [key: string]: number }와 같은 의미입니다. 하지만 Record는 더 간결하고 가독성이 좋습니다.
  • 제한된 키 타입: 키를 특정 문자열 리터럴로 제한하려면 Record<"key1" | "key2", number>처럼 사용할 수 있습니다.
const limited: Record<"width" | "height", number> = {
  width: 100,
  height: 200
  // depth: 50 // 에러: "depth"는 허용되지 않는 키
};
값 타입 일관성: 모든 값이 number여야 하며, undefined나 null도 허용되지 않습니다 (단, Record처럼 명시적으로 허용 가능).
